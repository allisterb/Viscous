# Screenshots

These images are referenced by the guides in [`docs/`](../). The files currently
in this folder are **placeholders** — replace each one with a real screenshot,
keeping the same filename so the documentation links keep working.

Suggested capture size: roughly 800×500 (or wider for tool windows). Use the
**dark** Visual Studio theme for consistency, and crop to just the relevant
window or dialog.

| File | What to capture |
|------|-----------------|
| `new-project-dialog.png` | The **Create a new project** dialog with the **Solidity Project** template selected. |
| `project-properties.png` | The project **Properties → General** page showing Compiler Version, EVM Version, and .NET Bindings Namespace. |
| `editor.png` | A `.sol` file open in the editor showing syntax highlighting, with a hover tooltip or completion list visible. |
| `build-errors.png` | The **Output** window and/or **Error List** after a build, showing a compiler error with file/line. |
| `static-analysis-window.png` | The **Solidity Static Analysis** tool window with Slither findings expanded under the impact folders. |
| `blockchain-explorer.png` | The **Blockchain Explorer** tool window with a network expanded to show Endpoints / Accounts / Contracts / Deploy Profiles. |
| `add-network-dialog.png` | The **Add EVM Network** dialog filled in. |
| `deploy-window.png` | The **Deploy Solidity Project** tool window with a contract and deploy profile selected. |
| `run-contract-dialog.png` | The **Run Contract** dialog showing the contract balance and function buttons. |
| `dotnet-bindings.png` | Solution Explorer (or a folder view) showing the generated `bindings` folder, or a generated C# binding file open. |
